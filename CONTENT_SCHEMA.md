# 内容模型规范

本文件是博客内容组织的唯一规范。定位、分类、系列、文章模板都以这里为准。

## 品牌与定位

- 品牌名：`Muzig`
- 规范域名：`https://muzig.io`
- 一句话定位：`面向开发者的 AI 工程深度博客，聚焦 Agent 工具链、MCP、Go 工程与编程语言设计。`

## 字段职责

- `categories`：主题域，用来回答“这篇文章属于哪个知识板块”
- `series`：系列归属，用来回答“这篇文章在哪条连续内容线里”
- `articleType`：文章形式，用来回答“这篇文章是怎么写的”
- `tags`：检索关键词，用来回答“这篇文章涉及哪些产品、协议、概念”

不要再用 `categories` 表达“技术深度 / 技术文章 / 技术思考”这类写法，它们属于文章形式，不属于主题分类。

## 允许的主题分类

- `AI工程`
- `Agent工具链`
- `MCP`
- `Go工程`
- `编程语言`
- `LLM系统`
- `软件架构`

## 推荐系列

- `AI 工程观察`
- `Claude Code 实战`
- `OpenClaw 深度系列`
- `MCP 技术系列`
- `Go 工程实践`
- `编程语言原理`
- `LLM 系统拆解`
- `架构拆解`

## 推荐文章形式

- `深度解析`
- `实战教程`
- `入门指南`
- `架构拆解`
- `对比分析`
- `方法论`
- `观点评论`
- `故障排查`

## 标签规则

- 优先使用产品名、协议名、语言名、技术概念
- 保持大小写和命名一致，例如 `OpenClaw`、`Claude Code`、`MCP`、`Go`
- 避免同义重复，例如不要同时使用 `AI Agent` 和 `Agent`
- 避免把栏目名塞进标签，例如不要把 `技术深度` 当标签

## Front Matter 示例

```toml
+++
date = '2026-04-22T10:00:00+08:00'
draft = true
title = '文章标题'
description = '一句话说明文章解决的问题'
categories = ['Agent工具链']
series = ['OpenClaw 深度系列']
articleType = '实战教程'
tags = ['OpenClaw', 'Agent', '工作区设计']
+++
```

## 迁移约定

- 旧的 `技术深度 / 技术文章 / 技术思考 / 技术经济` 已停止作为分类使用
- 需要表达文章深浅、教程属性、评论属性时，统一使用 `articleType`
- `type` 是 Hugo 内置字段，会影响内容分区；不要拿它代替文章形式
